# 模块 01：自主感知（环境/状态全感知）

## 能力概述
实时毫秒级扫描环境和自身状态，状态变化立即感知，无延迟。

---

## 感知范围

### 1. 系统环境感知
| 感知项 | 说明 | 检测频率 |
|--------|------|----------|
| CPU 状态 | 使用率、温度、核心数 | 实时 |
| 内存状态 | 已用/可用、占用率、Swap | 实时 |
| 磁盘状态 | 容量、读写速度、健康度 | 每分钟 |
| 网络状态 | 连接、延迟、带宽、IP | 实时 |
| 电源状态 | 电量、充电状态、功耗 | 实时 |
| 进程状态 | 运行中进程、资源占用 | 实时 |
| 系统权限 | 当前用户、sudo 权限、访问限制 | 启动时 |
| 时间 | 系统时间、时区、时戳 | 实时 |
| 地理位置 | GPS/WiFi 定位、IP 地理信息 | 按需 |

### 2. OpenClaw 自身感知
| 感知项 | 说明 | 检测频率 |
|--------|------|----------|
| 运行状态 | 活跃/空闲/错误 | 实时 |
| 技能加载 | 已加载技能列表、版本 | 启动时 |
| 内存占用 | 当前会话内存使用 | 每分钟 |
| 报错日志 | 错误信息、堆栈、时间 | 实时捕获 |
| 任务进度 | 进行中任务、完成度 | 实时 |

### 3. 外部界面感知
| 感知项 | 说明 | 检测方式 |
|--------|------|----------|
| 电脑桌面 | 桌面内容、图标、窗口 | 截图识别 |
| 软件窗口 | 窗口标题、位置、大小、状态 | 窗口 API |
| 网页界面 | DOM 结构、元素、状态 | 浏览器 API |
| 图像识别 | 按钮、输入框、文本、图标 | OCR+CV |

### 4. 任务环境感知
| 感知项 | 说明 | 触发条件 |
|--------|------|----------|
| 已执行任务 | 历史记录、结果、耗时 | 任务完成时 |
| 未完成任务 | 待办列表、优先级、截止 | 持续跟踪 |
| 外部指令 | 用户输入、API 调用、事件 | 实时监听 |
| 系统事件 | 通知、告警、定时任务 | 事件驱动 |

---

## 感知实现方式

### 系统层感知（调用脚本）
```bash
# CPU/内存/磁盘
python scripts/perception_monitor.py --system

# 网络状态
python scripts/perception_monitor.py --network

# 进程列表
python scripts/perception_monitor.py --processes
```

### 界面层感知（浏览器/桌面）
- 使用 `browser` 工具进行页面快照
- 使用 `canvas` 工具进行界面截图
- 使用 `nodes` 工具获取设备状态

### 任务层感知（内部状态）
- 读取 `memory/YYYY-MM-DD.md` 获取任务历史
- 检查 `temp/` 目录获取进行中任务
- 监听 session 消息获取新指令

---

## 感知数据处理

### 数据格式
```json
{
  "timestamp": "2026-03-14T18:31:00+08:00",
  "category": "system|self|interface|task",
  "items": [
    {"key": "cpu_usage", "value": 45.2, "unit": "%"},
    {"key": "memory_free", "value": 8192, "unit": "MB"}
  ],
  "changes": ["cpu_usage increased by 10%"],
  "alerts": []
}
```

### 变化检测
- 对比上一次感知结果
- 标记显著变化（阈值可配置）
- 触发思考模块处理变化

---

## 感知触发思考

感知到以下情况时，立即触发自主思考：

1. **系统异常**：CPU>90%、内存>95%、磁盘满、网络断开
2. **任务完成**：当前任务结束，需要决策下一步
3. **新指令到达**：用户输入新任务
4. **界面变化**：目标窗口/页面状态改变
5. **定时触发**：空闲时主动思考优化方案
