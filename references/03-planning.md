# 模块 03：自动计划（任务规划能力）

## 能力概述
自动拆解复杂任务为可执行子步骤，支持多级任务拆分，动态调整计划。

---

## 计划生成流程

```
输入：思考模块的决策输出
  ↓
任务拆解 → 生成子步骤
  ↓
顺序分配 → 确定执行顺序
  ↓
资源分配 → 分配时间/工具/权限
  ↓
输出：可执行计划
```

---

## 任务拆解规则

### 拆解粒度
| 任务复杂度 | 拆解层级 | 示例 |
|------------|----------|------|
| 简单任务 | 1 层 | "打开文件" → 直接执行 |
| 中等任务 | 2-3 层 | "写报告" → 收集资料→撰写→格式化 |
| 复杂任务 | 4+ 层 | "建站" → 需求→设计→开发→测试→部署 |

### 拆解原则
1. **原子性**：每个子步骤应该是可独立执行的最小单元
2. **可验证**：每个子步骤完成后应有明确的成功/失败判定
3. **可回滚**：关键步骤应有回滚方案
4. **依赖清晰**：明确步骤间的先后依赖关系

## 计划结构

### 标准计划格式
```markdown
# [任务名称] 执行计划
创建时间：YYYY-MM-DD HH:MM

## 目标
[一句话描述最终交付物]

## 步骤
- [ ] 步骤 1: xxx
  - 工具：xxx
  - 预计耗时：xxx
  - 前置条件：xxx
  - 验收标准：xxx
- [ ] 步骤 2: xxx
  ...

## 资源需求
- CPU: x%
- 内存：x MB
- 外部依赖：xxx

## 风险与应对
- 风险 1: xxx → 应对：xxx
- 风险 2: xxx → 应对：xxx

## 当前进度
正在执行：步骤 X
完成度：X%
```

### 计划文件位置
- 进行中计划：`temp/[任务名]-plan.md`
- 已完成计划：`archive/plans/[日期]-[任务名].md`

## 执行顺序分配

### 排序策略
| 策略 | 适用场景 | 说明 |
|------|----------|------|
| 依赖优先 | 有明确依赖关系 | 先执行前置任务 |
| 优先级优先 | 多任务并行 | 先执行高优先级 |
| 资源最优 | 资源受限 | 避免资源冲突 |
| 时间敏感 | 有截止时间 | 先执行紧急任务 |

### 并行/串行判断
```
可并行条件：
- 步骤间无依赖
- 资源不冲突
- 结果不互相影响

必须串行条件：
- 步骤 B 依赖步骤 A 的输出
- 共享同一资源（如文件锁）
- 逻辑上必须按顺序执行
```

---

## 资源分配

### 资源类型
| 资源 | 分配方式 | 说明 |
|------|----------|------|
| CPU | 百分比限制 | 避免占用过高影响系统 |
| 内存 | MB 限制 | 防止内存溢出 |
| 时间 | 超时设置 | 防止任务无限执行 |
| 工具 | 技能/脚本选择 | 选择最优工具 |
| 权限 | 权限级别 | 确保有执行权限 |

### 资源预估
```
基于历史数据预估：
- 类似任务的平均耗时
- 类似操作的资源消耗
- 当前系统可用资源

预留缓冲：
- 时间缓冲：+20%
- 内存缓冲：+30%
```

## 动态调整机制

### 调整触发条件
| 条件 | 调整动作 |
|------|----------|
| 环境变化 | 重新评估可行性，调整计划 |
| 步骤失败 | 分析原因，重试或修改方案 |
| 资源不足 | 降低优先级或暂停任务 |
| 新任务插入 | 重新排序，调整资源分配 |
| 用户干预 | 按用户指令调整计划 |

### 调整流程
```
检测到变化
  ↓
暂停当前执行
  ↓
重新分析场景（调用思考模块）
  ↓
生成新计划或调整现有计划
  ↓
记录调整原因到日志
  ↓
继续执行
```

## 计划执行脚本

使用 `scripts/auto_planner.py` 生成和管理计划：

```bash
# 生成新计划
python scripts/auto_planner.py create --task "任务描述" --output temp/task-plan.md

# 更新计划进度
python scripts/auto_planner.py update --plan temp/task-plan.md --step 2 --status completed

# 重新规划
python scripts/auto_planner.py replan --plan temp/task-plan.md --reason "环境变化"
```

## 计划验证

### 验证清单
- [ ] 目标是否明确可衡量
- [ ] 步骤是否完整无遗漏
- [ ] 依赖关系是否正确
- [ ] 资源分配是否合理
- [ ] 风险应对是否有准备
- [ ] 验收标准是否清晰

### 验证失败处理
- 缺失信息 → 返回思考模块补充分析
- 资源不足 → 调整计划或等待资源释放
- 风险过高 → 添加用户确认步骤
