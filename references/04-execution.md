# 模块 04：自动执行（任务落地能力）

## 能力概述
严格按照计划自动执行操作，无需人工点击/确认，执行过程实时反馈。

---

## 执行范围

### 1. 系统操作
| 操作类型 | 示例 | 工具 |
|----------|------|------|
| 文件操作 | 创建、读取、编辑、删除、移动 | `read`/`write`/`edit` |
| 目录操作 | 创建、遍历、清理 | `exec` |
| 进程管理 | 启动、停止、监控 | `exec`/`process` |
| 系统命令 | Shell 命令执行 | `exec` |

### 2. 软件启动
| 软件类型 | 启动方式 | 说明 |
|----------|----------|------|
| 命令行工具 | `exec` 直接调用 | 无需 GUI |
| 桌面应用 | `nodes` 调用设备应用 | 需要 GUI 支持 |
| 浏览器应用 | `browser` 打开网页 | 需要浏览器 |
| OpenClaw 技能 | `sessions_spawn` 调用子技能 | 复杂任务 |

### 3. 界面操作
| 操作类型 | 工具 | 说明 |
|----------|------|------|
| 页面导航 | `browser navigate` | 打开/跳转 URL |
| 元素点击 | `browser act click` | 点击按钮/链接 |
| 文本输入 | `browser act type` | 填写表单 |
| 截图识别 | `browser snapshot` | 获取页面状态 |
| 文件上传 | `browser upload` | 上传文件 |
| 文件下载 | `browser act waitForDownload` | 下载文件 |

### 4. 文件处理
| 处理类型 | 技能 | 说明 |
|----------|------|------|
| PDF 处理 | `pdf` skill | 创建、编辑、提取 |
| Word 处理 | `docx` skill | 文档生成、编辑 |
| Excel 处理 | `xlsx` skill | 表格操作 |
| 图片处理 | `canvas` skill | 截图、渲染 |

### 5. 命令执行
| 命令类型 | 工具 | 说明 |
|----------|------|------|
| 即时命令 | `exec` | 短命令，等待完成 |
| 长时命令 | `exec background` | 后台运行 |
| TTY 命令 | `exec pty=true` | 需要终端交互 |
| 进程管理 | `process` | 管理后台会话 |

## 执行模式

### 串行执行
```
步骤 1 → 步骤 2 → 步骤 3
```
- 适用：有依赖关系的步骤
- 保证：顺序执行，前一步完成后执行下一步

### 并行执行
```
步骤 1 ─┬
        ├→ 结果合并
步骤 2 ─┘
```
- 适用：独立无依赖的步骤
- 优势：提高效率，减少总耗时

### 混合执行
```
步骤 1 → 步骤 2 ─┬
                 ├→ 步骤 4
步骤 3 ─────────┘
```
- 适用：部分依赖、部分独立的复杂任务

## 执行反馈机制

### 实时反馈
| 反馈类型 | 内容 | 频率 |
|----------|------|------|
| 进度更新 | 当前步骤、完成度 | 每步骤完成 |
| 状态报告 | 成功/失败/进行中 | 实时 |
| 异常捕获 | 错误信息、堆栈 | 异常发生时 |
| 资源使用 | CPU、内存、时间 | 定期 |

### 反馈记录
```markdown
## 执行日志

[18:35:00] 开始执行步骤 1: 创建目录
[18:35:01] 步骤 1 完成：成功
[18:35:02] 开始执行步骤 2: 写入文件
[18:35:05] 步骤 2 完成：成功
[18:35:06] 所有步骤完成，任务成功
```

## 执行异常处理

### 异常类型
| 异常 | 处理策略 |
|------|----------|
| 命令失败 | 重试 3 次，失败则报告 |
| 超时 | 终止执行，记录原因 |
| 权限不足 | 请求提权或跳过 |
| 资源不足 | 暂停任务，等待释放 |
| 依赖缺失 | 安装依赖或报告 |

### 异常处理流程
```
捕获异常
  ↓
分类异常类型
  ↓
执行对应处理策略
  ↓
记录异常日志
  ↓
决定：重试/跳过/报告用户
```

## 执行脚本

使用 `scripts/auto_executor.py`（需创建）管理执行：

```bash
# 执行计划
python scripts/auto_executor.py run --plan temp/task-plan.md

# 执行单步
python scripts/auto_executor.py step --command "ls -la" --timeout 30

# 查看执行状态
python scripts/auto_executor.py status --task task-id
```

## 执行验证

### 验证方式
| 验证类型 | 方法 | 说明 |
|----------|------|------|
| 输出检查 | 检查命令输出 | 确认预期结果 |
| 文件检查 | 检查文件存在/内容 | 确认文件操作成功 |
| 状态检查 | 检查进程/服务状态 | 确认启动成功 |
| 界面检查 | 截图/快照对比 | 确认界面操作成功 |

### 验证失败处理
1. 记录失败原因
2. 尝试修复（调用修复模块）
3. 重试执行
4. 多次失败后报告用户

## 执行安全

### 安全原则
1. **破坏性操作确认**：删除、覆盖等操作需二次确认
2. **权限最小化**：使用最低必要权限执行
3. **可恢复性**：关键操作前备份
4. **日志完整**：所有操作记录日志

### 操作白名单
```
允许自动执行：
- 文件读取
- 目录创建
- 非破坏性命令
- 网页浏览

需要确认：
- 文件删除
- 系统配置修改
- 外部 API 调用
- 敏感数据访问
```
