# 模块 05：自动学习（进化能力）

## 能力概述
从行为、经验、优化、知识四个维度自动学习，学习结果自动固化，无需手动保存。

---

## 学习类型

### 1. 行为学习
**来源**：记录人工操作/成功执行的任务

**学习内容**：
- 操作流程：步骤顺序、工具使用
- 参数配置：常用参数、默认值
- 最佳实践：高效操作方式

**固化方式**：
```markdown
## 新学到的操作流程

任务：[任务类型]
步骤：
1. [步骤 1]
2. [步骤 2]
...

可复用场景：[适用场景]
最后更新：YYYY-MM-DD
```

### 2. 经验学习
**来源**：从失败/报错中总结原因

**学习内容**：
- 失败原因分析
- 避免方法
- 替代方案

**固化方式**：
```markdown
## 失败经验记录

任务：[任务类型]
失败原因：[具体原因]
避免方法：[如何避免]
替代方案：[备选方案]

记录时间：YYYY-MM-DD
```

### 3. 优化学习
**来源**：对比执行效率

**学习内容**：
- 更快的操作方式
- 更省资源的方案
- 更稳定的流程

**优化循环**：
```
执行任务 → 记录耗时/资源
  ↓
对比历史数据
  ↓
发现优化空间
  ↓
尝试新方案
  ↓
验证效果
  ↓
固化最优方案
```

### 4. 知识学习
**来源**：自动读取文档、规则、数据

**学习内容**：
- API 文档：接口用法、参数说明
- 业务规则：逻辑约束、验证规则
- 数据结构：Schema、字段含义

**固化方式**：
- 存入 `references/` 目录
- 更新 `MEMORY.md` 知识点
- 建立知识索引便于检索

---

## 学习触发条件

| 触发事件 | 学习动作 |
|----------|----------|
| 任务成功完成 | 记录成功流程到经验库 |
| 任务失败 | 分析原因，记录教训 |
| 发现更优方案 | 对比并更新最佳实践 |
| 读取新文档 | 提取知识点存入知识库 |
| 用户纠正操作 | 学习正确方式，覆盖旧认知 |

---

## 学习记录结构

### 学习日志格式
```markdown
# 学习日志 YYYY-MM-DD

## 新技能
- 技能名：[名称]
- 来源：[任务/文档/用户]
- 内容：[简述]

## 经验教训
- 场景：[什么情况下]
- 问题：[遇到什么问题]
- 解决：[如何解决]
- 避免：[以后如何避免]

## 优化发现
- 原方案：[原来的做法]
- 新方案：[优化后的做法]
- 提升：[效率提升多少]
```

### 学习存储位置
- 短期学习：`memory/YYYY-MM-DD.md`
- 长期知识：`MEMORY.md`
- 技能库：`~/.openclaw/skills/` 新增/更新技能

---

## 学习验证

### 验证方式
| 验证类型 | 方法 | 说明 |
|----------|------|------|
| 回放验证 | 重复执行学习的操作 | 确认流程正确 |
| 对比验证 | 与历史结果对比 | 确认一致性 |
| 边界验证 | 测试边界条件 | 确认鲁棒性 |

### 验证失败处理
- 重新学习：分析验证失败原因
- 标记存疑：标记该知识点需要人工确认
- 回滚：恢复到学习前的状态

---

## 学习脚本

使用 `scripts/auto_learner.py`（需创建）管理学习：

```bash
# 记录学习
python scripts/auto_learner.py record --type success --task "任务描述" --log temp/task-log.md

# 提取知识
python scripts/auto_learner.py extract --source doc.md --output references/knowledge.md

# 查询学习历史
python scripts/auto_learner.py query --keyword "关键词" --limit 10
```

| 优先级 | 学习类型 | 说明 |
|--------|----------|------|
| P0 | 失败教训 | 避免重复犯错 |
| P1 | 用户纠正 | 符合用户期望 |
| P2 | 效率优化 | 提升执行速度 |
| P3 | 新知识 | 扩展能力边界 |

## 学习固化机制

### 自动固化
```
学习完成
  ↓
格式化学习内容
  ↓
写入对应文件
  ↓
更新索引
  ↓
验证写入成功
```

### 定期整理
- 每天：整理当日学习到 `memory/YYYY-MM-DD.md`
- 每周：提炼重要知识到 `MEMORY.md`
- 每月：归档旧知识，保持知识库精简
