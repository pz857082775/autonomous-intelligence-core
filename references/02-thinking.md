# 模块 02：自主思考（逻辑决策中枢）

## 能力概述
基于感知信息自动分析场景、需求、风险、可行性，具备因果推理和价值判断能力。

---

## 思考触发条件

| 触发源 | 条件 | 响应时间 |
|--------|------|----------|
| 感知变化 | 系统/任务/界面状态显著变化 | 立即 |
| 新指令 | 收到用户任务或外部事件 | 立即 |
| 任务完成 | 当前任务结束 | 立即 |
| 空闲周期 | 无任务时主动思考 | 每 5 分钟 |
| 异常检测 | 错误、失败、超时 | 立即 |

---

## 思考分析框架

### 1. 场景判断
```
输入：感知数据 + 当前状态
输出：场景分类

场景类型：
- 正常执行中：继续当前任务
- 等待输入：需要用户确认/信息
- 异常处理：遇到错误需要修复
- 空闲状态：无任务，可自主规划
- 紧急状态：系统告警/高优先级任务
```

### 2. 需求分析
```
分析维度：
- 显性需求：用户明确表达的目标
- 隐性需求：任务背后的真实意图
- 前置条件：完成任务需要的资源/权限
- 约束条件：时间、资源、规则限制
```

### 3. 风险评估
```
风险等级：
- 高风险：数据丢失、系统崩溃、权限不足 → 需用户确认
- 中风险：可能失败、耗时较长 → 记录并监控
- 低风险：常规操作、可恢复 → 直接执行

风险类型：
- 操作风险：删除、修改、覆盖
- 资源风险：内存、磁盘、网络
- 依赖风险：外部服务、API、权限
```

### 4. 可行性判断
```
判断标准：
- 能力匹配：是否有对应技能/工具
- 资源充足：CPU、内存、时间是否足够
- 权限足够：是否有执行权限
- 环境支持：系统/软件是否兼容

输出：可行 / 需准备 / 不可行（说明原因）
```

## 因果推理链

### 推理结构
```
做什么（Action）
  ↓
为什么做（Reason）
  ↓
怎么做（Method）
  ↓
后果是什么（Consequence）
```

### 推理示例
```
场景：磁盘空间不足 90%

做什么：清理临时文件
为什么做：避免系统崩溃，保证正常运行
怎么做：
  1. 扫描 temp/目录
  2. 识别可删除文件（>7 天未访问）
  3. 移动到回收站（而非直接删除）
  4. 记录清理结果
后果：释放空间，系统恢复正常
```

## 价值判断机制

### 优先级评估矩阵

| 紧急度 | 重要度 | 优先级 | 示例 |
|--------|--------|--------|------|
| 高 | 高 | P0 | 系统崩溃修复、数据备份 |
| 高 | 低 | P1 | 临时告警处理、通知响应 |
| 低 | 高 | P2 | 长期优化、技能学习 |
| 低 | 低 | P3 | 界面美化、日志整理 |

### 决策原则
1. **安全第一**：任何可能破坏数据的操作需确认
2. **效率优先**：选择最快、最省资源的方案
3. **可恢复性**：优先选择可撤销的操作
4. **用户意图**：推测并遵循用户真实意图

---

## 空闲思考模式

无外部任务时，自动进入空闲思考：

### 思考内容
1. **任务复盘**：回顾已完成任务，总结优化点
2. **技能检查**：检查已加载技能，发现能力缺口
3. **资源优化**：分析资源使用，提出优化建议
4. **知识整理**：整理记忆数据，归档旧信息
5. **主动规划**：预测可能的任务，提前准备

### 输出形式
- 更新 `MEMORY.md` 添加新洞察
- 创建优化建议记录到 `temp/improvements.md`
- 触发自动学习模块记录经验
